What we want

Let me use ITRANS-to-Devanagari as an example. In the spirt of Test-Driven Development (although this is being written after the code :P), here are a few tests:
  • When the user types 'k', we need to see the string '&ka;&hal;', which is Unicode two characters: &ka;+&hal;
  • When the user types 'ka', we need to see the string '&ka;', which is one Unicode character.
  • When the user types 'kai', we need to see the string '&ka;&ai;', which is two characters: '&ka;'+'&ai;'.

How it works

Javascript's canonical data structure -- called Object -- is an associative array (also known as hash table, map, etc.), so we use it.

Bugs

  • It parses the whole string each time you type a letter. I expect that it will be used only for transliterating short pieces of text (at a time), so I do not view this as an urgent issue.
  • It converts all CONSONANT+VIRAMA+VOWEL occurrences to CONSONANT+VOWEL.
  • If you type "kaa" into Harvard-Kyoto you get [ka]+[a], but it is transliterated into ITRANS as "kaa" so if you then go to ITRANS and type something, the original Harvard-Kyoto changes to "kA" as well.
